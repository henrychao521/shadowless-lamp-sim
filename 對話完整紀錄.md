## 一、專案概觀

「無影燈光學模擬器」是一套給國中／高中生活科技課程使用的互動教具網站，
以 2D（Canvas 即時照度計算）與 3D（Three.js 物理光照）兩種視圖，演示外科無影燈
如何透過多光源、反射穹頂等設計消除手術視野中的陰影。技術組成：

- 純前端靜態網站（vanilla HTML / CSS / JS），無建置流程。
- 3D：Three.js（CDN）+ 後處理 Bloom + IES 光域檔。
- 圖表：Chart.js（CDN）繪製照度剖面。
- PWA：Service Worker 離線快取 + manifest 可安裝。
- 衍生頁：`optics-reciprocity.html`（光路逆行互動演示）、`操作介紹.pdf`（A4 繁中操作手冊）。

設計重點隨開發推進，從「3D 真實感」逐步轉向 **手機操作優化** 與 **光學原理正確性**。

### 關鍵工程教訓
- **deploy.yml 使用硬編碼檔案清單**：任何要上線的新檔案必須同時加到三處（cp 至 /tmp/site、cp 至 gh-pages、git add），否則 workflow 綠燈卻仍 404（Phase 54／56 教訓）。
- **.gitignore 勿用 `*.pdf` 全域排除**：會把自製「操作介紹.pdf」在兩分支靜默略過，改為僅排除廠商手冊檔名。
- **CDN 傳播延遲**：部署成功後線上 CDN 可能延遲數十秒，立即 curl 可能取得舊版，須稍後重驗；`git show origin/gh-pages:檔案` 為權威來源。
- **exFAT 卷宗 git 注意事項**：macOS 複製會產生 AppleDouble `._` 檔導致 git「non-monotonic index」（`find . -name '._*' -delete`）；exFAT 不保留執行位元（`git config core.fileMode false`）。
- **資產快取破壞**：`index.css?v=N` token 與 SW `CACHE_NAME` 每次改版遞增，確保使用者取得最新版本。

## 模型的核心問題：2D vs 3D

**你觀察到的「邊緣低於 50%」，問題根源不是醫學判斷，而是模型本身就是一個嚴重的簡化**。

### 模型做了什麼（現況）

```
LED-1  LED-2  LED-3  ... LED-n      ← 所有 LED 排在同一條直線上
              ↓↓↓↓↓
         [ 模擬頭部 ]
              ↓↓↓↓↓
════════════════════════════════     ← 工作平面（照度分佈圖）
```

這是一個 **2D 截面模型**：LED 是沿著一條弧線排列（只有左右），沒有前後。

### 真實無影燈是什麼（3D）

```
         ● ● ● ●
       ●         ●
      ●    LAMP   ●     ← LED 排成環狀（或多環）
       ●         ●
         ● ● ● ●
```

真實的無影燈 LED 排成 **360° 的環形或碗形**。這意味著：
- 某顆 LED 被頭部從「正前方」遮擋 → 後方的 LED 補光
- 左側被遮擋 → 右側、前側、後側都還在打光
- **三維交叉補光，影子幾乎消除**

---

## 你觀察到的「邊緣 < 50%」的真正原因

現行模型的照度圖在邊緣出現低值，是因為：

| 位置 | 2D 模型顯示 | 真實 3D 狀況 |
|------|------------|-------------|
| 中心點（x=0） | ~70–100%，較準確 | 幾乎 100%（多環補光） |
| 邊緣（x=±15~20cm） | 可能 <50%（光線角度不足） | 實際上更接近 80–90%（環形補光） |
| 遮擋正下方 | 明顯低谷，合理 | 低谷存在但比 2D 淺得多 |

**邊緣低值的成因：** 在 2D 中，只有「角度剛好打得到 x=±20cm」的 LED 才能貢獻，且部分被遮擋。在 3D 環形燈中，前後方向的 LED 同樣可以打到邊緣，所以邊緣照度反而很好。

---

## IEC 60601-2-41 的實際規範 vs 模型的誤差

**真正的 IEC 60601-2-41** 規定：
1. 中心照度 E_c ≥ 40,000 lux（絕對值）
2. 在定義的外科手術野（通常直徑 20cm 圓形）內，**任何點**的照度 ≥ 50% × E_c

**現行模型的 IEC 計算只看 x=0 的單點：**
```javascript
const iecPass = centerDilution >= 50;  // 只檢查最中心的一個 bin
```

這表示：如果遮擋物偏移（例如 x=5cm），中心點可能還在 80%，但偏移造成的陰影落在 x=-5cm 那個 bin，降到 30%，模型仍顯示「IEC PASS」——這是錯誤的。

## 多面反射式無影燈的光學原理

```
                   ● 中央單一 LED（或鹵素燈）
                  /|\
                 / | \
                /  |  \
   外環鏡片↗  /   |   \  ↖內環鏡片
           ↗    |    ↖
          ↗     |     ↖
─────────────────────────── 工作平面
  從「斜角」  從「垂直」  從「斜角」
  照射工作區  照射工作區  照射工作區
```

每一片小鏡面都把**同一個光源**的光，反射到工作平面上的**不同方向角**。等效於：
- 中央光源 → 被分裂成幾百個「虛擬點光源」
- 每個虛擬光源對應不同的入射角
- 手術醫師的手遮住某一角度的光 → 其他角度的光仍然照亮遮蔽區域

這個設計稱為 **多區段聚焦反射器（Multi-Zone Reflector）**，是比 LED 陣列更早出現的經典無影燈設計。

---

## 對比：兩種設計哲學

| 特性 | DomeLux 多面反射式 | 現行模擬（LED 陣列式） |
|------|-------------------|----------------------|
| 光源數量 | 1個（或少數幾個） | 多個 LED 模組（5–50+） |
| 無影原理 | 單源 → 鏡片分裂為多虛擬源 | 多點光源各自投射 |
| 有效角度 | 極廣（0°–60°，由鏡片傾角決定） | 受 LED 排列弧度限制 |
| 照度均勻性 | 鏡片數越多越均勻 | LED 數越多越均勻 |
| 主要遮擋源 | 中央把手（手術醫師握持） | 無固定遮擋 |

## 照片中特別值得注意的細節

**中央把手遮擋問題（照片 2、4）：**
```
          ══════ ← 把手（醫師握持）
         /      \
        /  鏡片   \
       /  ████████ \
      │ ██████████  │
       \ ████████  /
        \        /
```

把手本身就是光源前方最大的遮擋物，每台無影燈都必須在設計上讓把手周圍的鏡片「補償」這個固定陰影。這是多面反射設計中非常聰明的地方——把手遮擋的角度，恰好由旁邊鏡片的反射角填補。

## 這對我們的模擬代表什麼？

### 現行模擬的問題

我們的模型假設是「多個 LED 在弧形上排列」，但 DomeLux 實際是「多面反射式」。兩種設計的**物理原理相同**，但：

1. **虛擬光源位置不同**：反射式的虛擬源分布比 LED 陣列更密集、角度範圍更廣
2. **遮擋物的形狀不同**：真實燈具的遮擋源包含把手、支架，不只是球形頭部
3. **模型精確度**：多面反射式在 3D 上的均勻性比 2D LED 弧形模型更好

### 建議的模擬改進方向

**選項 A（說明層）：** 在模擬器加一段說明

- 把光源改為中央單點

## 可以類比的核心：凹面鏡的「焦點 ↔ 平行光」性質

牛頓式反射鏡與多面反射式無影燈，**共用同一個物理基礎**：

- **拋物面凹鏡**：放在焦點的點光源，反射後會變成平行光；反過來，平行入射光會聚到焦點。
- 這正是**光路可逆**（reversibility of light）的展現。

所以最精準的一句話是：

> **無影燈的反射碗 ≈ 牛頓望遠鏡的主鏡「反過來用」。**
> 望遠鏡是「平行光進來 → 聚到焦點成像」；手術燈是「焦點放燈泡 → 反射成（近似）平行/會聚光投射出去」。兩者是同一塊鏡面、相反方向的**互逆光學系統**。

## 但有三個關鍵差異，類比時務必點出

| 面向 | 牛頓式反射鏡（望遠鏡） | 多面反射式無影燈 |
|---|---|---|
| **光的方向** | 收光、**成像**（光線向內聚到焦點） | 出光、**投射照明**（光線從焦點向外） |
| **設計目的** | 要**一個銳利焦點**→ 主鏡是**一整面光滑拋物面** | 要**刻意失焦、散開**→ 把鏡面切成**數千片微鏡格**，各投一塊重疊光斑來補影 |
| **次鏡（45°斜鏡）** | 牛頓式的**招牌特徵**：一片平面斜鏡把光導向側邊目鏡 | **沒有對應元件**，光直接從主鏡射向術野 |

## 結論：類比要怎麼用才精準

1. **相同**：凹面鏡＋焦點的幾何（拋物面 focus↔parallel）、光路可逆。
2. **最像的其實是「探照燈／車頭燈反射鏡」**——光源在焦點、光向外射，方向與無影燈一致；而「探照燈＝望遠鏡倒過來用」這句話可同時把三者串起來。
3. **真正的巧思在「多面」**：望遠鏡追求單一銳焦（光滑鏡面），無影燈反其道而行，用**微鏡格陣列把銳利光束打散成柔和光池**來消除陰影——這層差異剛好能讓學生體會「同樣的凹面鏡，因設計目的不同而長相迥異」。
