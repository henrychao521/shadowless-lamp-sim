🛠️ 開發階段紀錄

Phase 0 — 初始研究與分析 完成

2026-05-26 · commit d0a4b9b 初始版本

對現有 3D 介面進行全面分析，識別出四大改善方向：

  • 光學物理：12 顆 SpotLight 配置、IES 光形、平方反比衰減
  • PBR 材質：所有物件使用純色 MeshStandardMaterial，無貼圖導致「塑膠感」
  • 場景構圖：缺少牆面、天花板，SSAO 無空間發揮
  • 後處理管線：Bloom 在 SSAO 前（錯誤順序）

Phase 1 — 後處理修正 + 材質升級 完成

2026-05-26 · commit 15fbe6f
  • 修正後處理順序：Render → SSAO → Bloom → Output
  • Bloom 調整：strength 1.0 → 0.35，radius 0.5 → 0.8（柔光感）
  • 加入 RoomEnvironment IBL：程序生成室內環境光，不需外部 HDR 檔
  • AmbientLight → HemisphereLight：天花板暖光 + 地板冷反射
  • 材質升級 MeshPhysicalMaterial：
    • 地板：clearcoat 0.5（醫療磁磚光澤）
    • 手術台：clearcoat 1.0（拋光不鏽鋼）
    • 覆布：sheen 1.0 + sheenColor（手術布料微光澤）
    • 傷口：clearcoat 0.8, roughness 0.3（濕潤組織）
    • 皮膚：transmission 0.08 + attenuation（次表面散射近似）
    • 燈頭：clearcoat 0.7（烤漆金屬感）
  • 加入手術房：BoxGeometry BackSide 牆面 + 天花板
  • shadow.normalBias = 0.02：消除表面自陰影噪點

Phase 7 — URL 狀態分享 + 滑桿數值動畫 完成

2026-05-26 · 自動 Loop 繼續
  • URL Hash 狀態編碼（simulation.js）：
    每次 input 事件後呼叫 encodeHash()，將六個參數編碼為 #h=100&ox=0&oy=50&or=10.5&n=25&s=6.0 格式寫入 window.location.hash（使用 history.replaceState 不產生瀏覽記錄）。
    頁面載入時 restoreFromHash() 讀取 hash 並還原各滑桿數值，讓用戶可複製 URL 分享特定設定。
  • 滑桿數值閃光動畫（index.css + simulation.js）：
    新增 @keyframes value-pulse：0% 藍色 → 35% 純白 + glow → 100% 藍色， duration 0.45s ease-out。每次 input 事件對對應的 val_* span 重設動畫 （先 classList.remove + reflow + classList.add）。 手機用戶移動滑桿時，數值有即時閃光提示，不需低頭看 label。

Phase 6 — 3D 載入提示 + Tab 自動捲動 完成

2026-05-26 · 自動 Loop 繼續
  • 3D Canvas 載入 Spinner（index.html + index.css + simulation3d.js）：

Phase 5 — 動畫精確化 + 圖表手機響應式 完成

2026-05-26 · 自動 Loop 繼續
  • 控制面板動畫精確化（index.html JS）：
    取代 max-height:9999px 反模式，改用 panel.scrollHeight 動態測量實際高度， 展開時設精確目標值 + transitionend 後解除限制； 收合時雙 requestAnimationFrame 確保瀏覽器正確執行 max-height 0 的過渡。 easing 從 ease 改為 cubic-bezier(0.4,0,0.2,1)（Material Design 標準）。

    • 手機：縮短 dataset 標籤（避免圖例過長截斷）
    • 手機：legend boxWidth 縮至 12、padding 8

Phase 4 — CSS Cascade 修正 + 行動 GPU 效能 完成

2026-05-26 · 自動 Loop 繼續 · commit （本次）

發現並修正 Phase 3 中潛藏的 CSS 架構 Bug，同時加入行動裝置 3D 效能優化。

  • 🐛 修正 CSS 層疊順序 Bug： index.css 中三個 @media 區塊原來放在 基礎樣式之前， 導致同優先級的桌機樣式（後定義）覆蓋手機響應式樣式。 重組 CSS：所有 @media 移至檔案末尾，確保手機樣式正確生效。 受影響的 class：.controls-panel、.tab-btn、 input[type="range"]、.canvas-3d、 .card-header、.legend、.view-card 等。
  • 新增 --accent-amber CSS 變數：補上 #f59e0b，修正智慧補償 label 顏色缺失。
  • canvas touch-action: none：防止 iOS 在 3D 畫布上觸發頁面滾動，確保 pinch-zoom 正常。
  • 手機橫向（landscape）斷點：新增 @media (max-width:768px) and (orientation:landscape)，canvas 高度調整為 70vw / max 380px。
  • 行動裝置 3D 效能（simulation3d.js）：
    • UA + viewport 偵測 isMobile
    • 行動：停用 SSAO（ssaoPass.enabled = false），避免環境遮蔽耗盡 GPU
    • 行動：Bloom strength 0.35 → 0.25（減輕後處理負擔）
    • 行動：antialias: false（WebGL 抗鋸齒改由手機 GPU 硬體處理效率更高）
    • 行動：shadow map 從 PCFSoftShadowMap → PCFShadowMap（減少 shadow 採樣）
    • 行動：pixelRatio 上限 2（Retina 3x → 限 2x，渲染面積降 11%）
    • OrbitControls 明確設定 touches.ONE = ROTATE, TWO = DOLLY_PAN

控制面板動畫升級（index.html）：
max-height: 9999px 是 CSS 動畫的反模式 — 瀏覽器需要從 0 補間到 9999px，即使實際內容只有 400px，導致開頭一段時間沒有動畫效果，最後瞬間「彈出」。
新方法：JS 使用 panel.scrollHeight 測量實際高度，openPanel() 設定精確目標高度，transitionend 事件後清除固定值（允許面板內容變化）；closePanel() 先固定高度再雙幀歸零，確保過渡動畫正確觸發。easing 改為 cubic-bezier Material Design 曲線。
